# Groups

A group is a named collection of users within an **Organization Unit (OU)**. Use groups to organize users and assign roles to multiple users at once rather than assigning roles individually.

## Groups and Organization Units

Every group belongs to exactly one OU. This scoping has two key effects:

- **Groups are OU-scoped** — a group can only contain users from its own OU. Users in other OUs cannot be members of that group.
- **Role assignments are scoped** — roles assigned to a group apply within the context of that group's OU. Users in a child OU inherit group membership when groups are nested.

Groups can also contain other groups as members, allowing you to build nested groupings. Adding a group as a member transitively includes all users in that group.

Because user types are also tied to OUs, users in a group naturally share the same schema and registration rules. For example, a group under the **Customers** OU contains only Customer-type users.

:::note
A group's OU cannot be changed after creation. If you need to move a group to a different OU, delete it and recreate it under the correct OU.
:::

## Create a Group

1. Navigate to **Groups** in the Thunder Console and click **New Group**.
2. Enter a **Name** and an optional **Description**.
3. Select the **Organization Unit** the group belongs to.
4. Optionally add **Members** — search for users or existing groups to include.
5. Click **Create Group**.

## Update a Group

1. Open the group from the **Groups** list.
2. Edit the **Name** or **Description**.
3. Click **Save**.

## Delete a Group

1. Open the group from the **Groups** list.
2. Click **Delete** and confirm.

:::warning
Deleting a group is permanent. Members are not deleted — only the group itself and its role assignments are removed.
:::

## Manage Members

### Add Members

1. Open the group from the **Groups** list.
2. Click **Add Members**.
3. Search for the users or groups you want to add and select them.
4. Click **Add**.

### Remove Members

1. Open the group from the **Groups** list.
2. Locate the member you want to remove.
3. Click **Remove** next to the member and confirm.

## Assign Roles to a Group

Roles assigned to a group apply to all members of that group. This is the recommended way to grant permissions to a set of users instead of assigning roles individually.

To assign a role to a group, see [Roles](/guides/guides/roles).
